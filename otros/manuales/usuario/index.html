Introducción

  MVCRivero es un framework desarrollado bajo arquitectura mvc el cual posea una separación del modelo, negocio y vistas comunicándose entre sí a través del controlador. MVCRivero posee ciertas clases las cuales ayudarán en el desarrollo, además de su fuerte que es la permisología donde se puede usar en su modalidad de un solo usuario para la conexión a base de datos o cada usuario del sistema representará uno en base de datos, al usar el framework usted está brindándole seguridad a sus datos.

  Este software es presentado como proyecto de grado para ing. Informática realizado por Geraldine Castillo, Jhonny Vielma y Johán Álamo como tutor acadénico en el I.U.T. Dr Federico Rivero Palacio en Caracas- Venezuela.


Requisitos mínimos

  Tanto para instalar como para usar el MVCRivero se necesitan ciertos requerimientos que serán mencionados a continuación.

PHP 5 o superior.
Servidor apache.
Base de datos postgreSQL versión 9.1.
Clase de conexión a base de datos PDO.
Conector de php a postgres.
Tener los errores de PHP activados(opcional).
Poseer permisos de escritura y lectura en WWW.
Poser servidor en linux(preferiblemente).



Al instalar

  Al momento de instalar el software este comprobará que los requisitos estén cumplidos al 100% de resto no procederá a la instalación.


Administración de acciones

  MVCRivero cuenta con una vista en la cual permite a un usuario administrar las acciones(agregar, modificar, listar y eliminar) de los módulos del sistema. Si son usuarios de base de datos permitirá asignar los permisos que posee esta acción en ciertas tablas.

¿Qué son acciones?

   Las acciones son privilegios a funcionalidades del sistema como por ejemplo la acción agregarUsuario, permite crear usuarios.

Sin usuarios de base de datos

  Cuando la configuración elegida es sin usuarios de base de datos no se tendrá la opción de colocar permisos de una acción sobre algunas tablas ya que solo existe un solo usuario para la conexión a la base de datos.

¿Cómo acceder?

   Para poder acceder a la pantalla donde se encuentra la administración de acciones, el usuario debe poseer mínimo el permiso o privilegio de listar acciones y de poseerlo en el menú superior se encontrará la opción de permisología, se pasa el cursor en esta opción y se desplegará una lista en la cual se encontrará administrar acciones.

Administración de tabla

  MVCRivero cuenta con una vista en la cual permite a un usuario administrar las tablas(agregar, modificar, listar y eliminar).   Estará disponible solo cuando la aplicación esté trabajando con usuarios de base de datos.

¿Qué son tablas?

   Son todas las tablas que existen en la base de datos de la aplicación.

Tipos de tabla

   Existen 3 tipos de tablas:

Tablas agregadas: Son aquellas que existen como tablas en base de datos y están agregadas en per.t_tabla para su administración.

Tablas no agregadas: Son aquellas que existen en la base de datos pero no en per.t_tabla ya que no han sido cargadas.

Agregadas no existente en BD: Son aquellas que están cargadas en per.t_tabla pero no existe ninguna tabla en la base de datos que coincida con este nombre.

¿Cómo acceder?

   Para acceder a la pantalla donde se encuentra la administración de tablas el usuario debe poseer mínimo el permiso o privilegio de listar tablas, de poseerlo en el menú superior se encontrará la opción de permisología, se pasa el cursor sobre esta y se desplegará una lista en la cual se encontrará administrar tablas. también al modificar, agregar o clonar una acción se encuentra un botón (de una llave) de administración de tablas en el cual también se puede acceder.

Administrar usuario y roles

  


¿Qué son los usuarios?

   Son los que utilizarán el sistema, el framework al instalarlo creará un usuario administrador que será el creador de la aplicación, mediante este se permitirá crear otros usuarios.

¿Qué son los roles?

   Los roles son aquellos "usuarios" que se crean para cargarle una permisología que será utilizada como referencia para clonarsela a usuarios reales. No poseen conexión al sistema.

Usuarios vs roles

   Los usuarios pueden iniciar sesión en el sistema y los roles no pueden ya que ellos solo son referencia, un rol no posee permisos de iniciar sesión.

Usuarios vs rol

Usuarios de base de datos

   Si se está utilizando la aplicación con la configuración de usuarios de base de datos, por cada usuario que se cree en la aplicación se guardará uno en base de datos, no se tendrá la contraseña del usuario en ninguna tabla solo el nombre, tipo, campo1,campo2 y campo3 en per.t_usuario. Permitirá manejar el acceso que tendrá sobre la información.

Sin usuarios de base de datos

   Si se está utilizando la aplicación con la configuración sin usuarios de base de datos, el usuario solo se almacenará en la tabla per.t_usuario con su contraseña. Las conexiones a la base de datos serán con el administrador.

Pantalla administrar usuarios

¿Cómo acceder?

   Para acceder a la pantalla donde se encuentra la administración de usuarios se debe poseer mínimo el permiso o privilegio de listar usuarios, de poseerlo en el menú superior se encontrará la opción de permisología, se pasa el cursor sobre ella y se desplegará una lista en la cual se encontrará administrar usuarios.

Administrar privilegios

  


¿Qué son los privilegios?

   Son los permisos que tiene un usuario sobre una acción (leer qué es una acción). Por ejemplo agregarUsuario igual a true, es decir; puede agregar un usuario.

¿Para quá sirven?

   Los privilegios sirven para negar o dar el acceso de usuarios a funcionalidades de la aplicación.

Privilegios con usuarios de base de datos

   Cuando se administrán los privilegios con usuarios de base de datos, el usuario que se le otorgue una acción se le concederán los permisos en tablas que esta tenga asociada, si quita el privilegio sobre la acción revocará los permisos sobre las tablas.

Privilegios sin usuarios de base de datos

   Si se utiliza la aplicación con la configuración sin usuarios de base de datos, al momento de otorgarle un privilegio o acción a un usuario no se realizará ningún cambio en base de datos ya que solo existe la conexión del administrador. Solo puede utilizar el objeto permiso para asignar o revocar permisos sobre funcionalidades.

Usuarios con permiso de otorgar y revocar privilegios

   Cuando se tiene la configuración de usuarios de base de datos, los usuarios que tengan el privilegio de otorgar o revocar privilegios a otros pasarán a ser super usuarios de base de datos ya que solo estos son los que pueden realizar grant y revoque sobre tablas.

Pantalla de administrar privilegios

¿Cómo acceder?

   Para acceder a la pantalla donde se encuentra la administración de privilegios, el usuario debe poseer mínimo el permiso de administrar privilegios y en el menú superior se encontrará la opción de permisología, se pasa el cursor sobre esta y se desplegará una lista en la cual se encontrará administrar privilegios y darle click.

Acceder a administrar privilegios

Importante

   Al usar usuarios de base de datos y otorgar la acción de administrar privilegios este usuario pasará a ser un super user de base de datos con el mismo poder del administrador.

   Debe buscar los privilegios que le asignará al usuario o rol ya sea por el campo de búsqueda o scroleando en la lista, están representados con un swich que de estar negado se encuentra con la palabra off en rojo y de poseerlo se encuentra con la palabra on en verde, cuando se modifica un privilegio en su lado derecho del swich posee un * que representa el cambio que tiene ese privilegio, azul es que se mantiene, verde que se otorgará y rojo que se revocará, al haber localizado los permisos a otorgar se procede a cambiarlos de off a on arrastrando el swiches hacia la derecha hasta que diga on.

   Luego de haber otorgado los privilegios al usuario se procede a guardar los cambios en el botón azul al final que dice guardar cambios, para verificar, en el pie de la lista existen unos contadores los cuales llevan los permisos que se mantienen en azul, la cantidad de permisos a otorgar en verde y los que se revocaran en rojo. Revocar privilegios

  Esta opción permite revocar privilegios a usuarios para que no tengan acceso a funcionalidades del sistema, le explicaremos paso a paso como revocar un privilegio.

<

Paso 1(selección de usuario o rol)

   Debe seleccionar el usuario o rol al cual revocará los privilegios, ya sea mediante el cuadro de búsqueda o scroleando en la lista de usuarios que se encuentra en la pantalla, posicione el cursor sobre él y aparcerá un ícono de color verde en forma de lápiz al cual debe darle click para desplegar la pantalla de privilegios.

Icono administar privilegios a usuario
<

Paso 2(selecionar privilegios a revocar)

   Debe buscar los privilegios que le revocará al usuario o rol ya sea por el campo de búsqueda o scroleando en la lista, los privilegios están representados con un swich que de estar negado se encuentra con la palabra off en rojo y de poseerlo se encuentra con la palabra on en verde. Csuando se modifica un privilegio en su lado derecho posee un * que representa el cambio que tiene, azul es que se mantiene, verde que se otorgará y rojo que se revocará, se procede a cambiarlos de on a off arrastrándolos hacia la izquierda.

Clonar privilegios

  Esta opción permite clonarle a un usuario la permisología de otro.

<

Paso 1(seleccion de usuario o rol)

   Debe seleccionar el usuario o rol al cual le colocarán los privilegios del otro, ya sea mediante el cuadro de búsqueda o scroleando en la lista de usuarios que se encuentra en la pantalla, posicione el cursor sobre él y se aparecerá un ícono de color verde en forma de lápiz el cual debe darle click para desplegar la pantalla de privilegios.

Ícono administar privilegios a usuario
<

Paso 2(Buscar usuario a clonar)

   Luego se posiciona en el campo de búsqueda de clonar, introduce el usuario que se desea clonar su permisología, éste posee un autocompletar para buscar los usuarios y roles, se procede a darle click e inmediatamente se cargarán todos los privilegios .
